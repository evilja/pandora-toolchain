----- Command behaviour changes -----

/gitcode
- timeout should be 180 seconds.

/smartcode
- should not upload fonts.zip.

/get and /job 
- should be Rank 0

/attach, /detach and /init 
- should be Rank 1

/destruct
- should be Rank 2

/lstoken and /rmtoken
- should be Rank 3
- these commands should be created and they should do, in order:
A: List all tokens by their first 3 and last 3 characters, their label and if they are local, with a page selector if needed.
B: Remove a token by its label. Removes every token with the EXACT MATCH label.

/lsauth
- should be Rank 3
- this command should be created and it should do:
A: List all authenticated users as <@ID> in a rank level.

/gitsync
- should be Rank 3

/acixconfirm and /acixtemplate
- should be Rank 4
- their description should have "[BETA-TESTING]" at the beginning, without exceeding 100 characters.

/changerank
- should be Rank 4
- this command should be created and it should do:
A: Edit rank of a command, other than itself, to a desired level. 

/scrape
- should be removed.

----- API behaviour changes -----

POST /jobs/:id/cancel
- rate-limit should be lifted.

POST /gitsync
- should look for a PNwitch label, and reject if it is undetected.

GET /acix/search
- should accept a JSON instead of parameters like q=<text>.

POST /acix/publish
- should look for a PNwitch label, and reject if it is undetected.

----- Non-command behaviour changes -----

- A new authentication level, 4, should be created with name "witch.pandora"
- Ranks of commands should be stored in an env.pandora-like file instead of Rust source code.
