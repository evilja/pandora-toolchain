Pandora behavior inventory
==========================

This file lists the user-facing Discord commands, message/reaction entries, and
HTTP API routes implemented in the current codebase.

Access model
------------

- /help and /providers bypass allowlists.
- Other slash commands are checked against
  DB/config/global/environment/command_ranks.pandora. Missing entries are seeded
  from the built-in defaults. Higher ranks can run lower-rank commands.
- Rank 0, authorize.pandora: encode, pancode, probe, backup, backupall,
  gitcode, smartcode, merge, release, source, get, job.
- Rank 1, fansubber.pandora: attach, init, detach, !ts, font.
- Rank 2, admin.pandora: destruct, hearts, configure, edit, readmebase, addapi,
  translations, auth, remove.
- Rank 3, upper.pandora: gitsync, gentoken, lstoken, rmtoken, lsauth.
- Rank 4, witch.pandora: acixconfirm, acixtemplate, changerank.
- /auth and /remove additionally require the caller to have at least the target
  level rank.
- API routes under /api/v1 require Authorization: Bearer <token>. Non-GET/HEAD
  API requests are rate-limited per token except POST /jobs/:id/cancel. Tokens
  may be global or local (token|local|server_id). Local tokens force the
  effective server id for supported routes and are required for /api/v1/git/*
  routes. Token labels are read from the preceding ; comment line.

Common worker behavior
----------------------

- Discord worker jobs create a response message and most long-running encode
  jobs add a cross reaction for cancellation.
- API jobs use Frontend::Web and do not edit Discord messages.
- The worker queue declines a new full job when more than four jobs are already
  queued.
- Stages are Queued, Probing, Probed, Downloading, Downloaded, Encoding,
  Encoded, Uploading, Uploaded, Failed, Declined, Cancelled.
- On startup, the worker marks stale non-terminal jobs failed, removes
  pandora-job-* qBittorrent torrents, and clears the encode input cache.
- Probe jobs idle in Probed for 180 seconds, then archive if no follow-up uses
  them.
- A cross reaction creates a CANCEL sentinel in the job work directory. Only a
  job owned by the reacting user is matched. Forwarded API encode children are
  cancelled directly.

Discord slash commands
----------------------

/help
- Public. Sends an ephemeral interactive command guide. Select menu updates are
  owner-scoped; another user must run /help themselves.

/providers
- Public. Shows built-in download/encode support and configured provider state:
  Google Drive (server or global), Doodstream, LuluStream, Voe, Abyss,
  AnimeciX, AniSub, and GitHub/Forgejo persistence.

/encode torrent:<link> subtitle:<attachment> [preset] [concat]
- Rank 0. Downloads the subtitle attachment and queues JobType::Encode.
- The torrent value is classified as torrent URL, magnet, or Google Drive.
- Preset choices map to Standard, PseudoLossless, or Dummy. "gpu" currently
  maps to Standard in slash/API parsing. concat selects an intros.toml group.
- The pipeline downloads input, encodes with ffmpeg, then uploads.

/probe torrent:<link>
- Rank 0. Queues JobType::Probe. Downloads/probes a torrent and lists files for
  later /pancode or /backup selection.
- Google Drive inputs fail in the probe worker.

/pancode job_id:<probe> index:<file> subtitle:<attachment> [preset] [concat]
- Rank 0. Queues JobType::Pancode using a previous active Probed job and file
  index. The new job reuses the probe torrent metadata and attached subtitle.
- When a pancode child finishes, its parent probe job is archived.

/backup [torrent] [job_id] [index]
- Rank 0. Queues JobType::Backup.
- With torrent, downloads that torrent/Drive source and uploads the MKV only to
  Google Drive.
- With job_id, index is required and the source comes from a previous probe job.

/backupall torrent:<link>
- Rank 0. Queues JobType::BackupAll. Downloads a torrent and uploads every MKV
  to Google Drive.

/gitcode torrent:<link> subtitle_url:<url> [preset] [concat]
- Rank 0. Fetches subtitle bytes from subtitle_url with a 180 second HTTP
  timeout, rewrites GitHub blob URLs to raw.githubusercontent.com, then queues a
  normal encode job.

/smartcode episode:<n> [link] [preset] [concat]
- Rank 0. Requires the channel to be attached to an anime repo.
- Reads TL and optional TS from the episode folder. If link is omitted, reads
  the first non-empty, non-; line from SOURCE.md and strips a leading #.
- If TS is missing, runs pnass split-signs, uploads the sign-stripped TL and
  generated TS, then merges.
- During pnass merge, unused styles are pruned from TL and TS separately before
  duplicate TS style names are renamed.
- Runs pnass merge, uploads Release - <name> - E<NN>.ass, writes SOURCE.md when
  link was supplied, and queues a regular encode against the merged ASS.
- If pnass produces no Dialogue lines, smartcode fails before uploading the
  release ASS.
- If the channel has an AnimeciX template, the queued job carries pending
  AnimeciX publish metadata.

/merge episode:<n> [link]
- Rank 0. Runs the same smartcode merge/upload flow but does not queue an
  encode. Responds with repo, release path, source path, and warnings.

/release episode:<n>
- Rank 0. Requires an attached channel and existing release ASS.
- Reads Release - <name> - E<NN>.ass (or .zip), parses fonts, uploads
  episode/fonts.zip from DB/fontconfig/<server_id> and DB/fontconfig/global, and
  attempts AniSub upload when anisub is configured.

/source episode:<n> link:<source>
- Rank 0. Writes episode/SOURCE.md in the attached repo with "# <source>".
- Nyaa download/torrent links are normalized to /view/<id>. .gitkeep in the
  episode folder is removed after upload.

/get type:<Translation|Typeset> episode:<n>
- Rank 0. Returns a Forgejo download URL for the episode TL or TS file, checking
  .ass.zip before .ass.

/job type:<TL|TLC|TS> episode:<n> subtitle:<ass_or_zip> [commit]
- Rank 0. Requires the channel to be attached and episode in range.
- Accepts .ass or a .zip with exactly one root-level .ass. Unsafe zip paths and
  multiple/zero ASS files are rejected.
- Runs pnass through PNASS_JOB with --no-adv-parsing, --title, and --wrap-style.
  This reserializes the file through libkagami, drops unsupported/nonstandard
  sections such as Aegisub Project Garbage, and avoids advanced override-tag
  optimization so raw dialogue text and raw {...} blocks are preserved.
- Duplicate Script Info keys are parsed last-key-wins inside pnass, but /job
  always overrides Title. WrapStyle is forced only when server wrapstyle is 0,
  1, 2, or 3; otherwise pnass keeps the parsed file value.
- pnass opcode 4 line-length warnings are collected and shown in the embed.
- TL writes TL; TLC also writes the TL filename; TS writes TS.
- Output filename is "<TL|TS> - <safe anime name> - E<NN>.ass". Files larger
  than ASS_ZIP_THRESHOLD_BYTES are uploaded as .ass.zip and the alternate path
  is removed.
- Default commit messages are Translation, Edit, and Typeset. A custom commit is
  prefixed with [TL], [TLC], or [TS].

/attach mal:<url> repo:<url> [season] [tl] [tlc] [ts] [qc]
- Rank 1. Fetches MAL/JIKAN metadata, validates Forgejo config, attaches the
  current channel to an existing repo, bootstraps episode folders/.gitkeep, and
  writes channel meta.
- Uses server base.md as README template when present.
- Counts existing episode folders, stores credits/season, and attempts to rename
  the Discord channel to the anime name.
- On attach only, any episode folder containing exactly one .ass is renamed to
  the standard TL filename for that episode.
- Refuses to reattach the channel to a different MAL id.

/init mal:<url> [season] [tl] [tlc] [ts] [qc]
- Rank 1. Same bootstrap behavior as /attach, but creates a new public Forgejo
  repo first using the MAL slug with hyphens replaced by underscores.

/detach
- Rank 1. Removes the channel meta.toml attachment. The Forgejo repo is left
  untouched.

/destruct
- Rank 2. Deletes the attached Forgejo repo and removes the channel attachment.

/configure language:<EN|TR|JP> [forgejo] [api_key] [gdrive_*] [wrapstyle]
- Rank 2. Writes DB/config/<server>/meta.pandora.
- Records current channel as announcement channel. forgejo must be http(s), or
  omitted/empty to unset. Google Drive server credentials must be all present or
  all empty. wrapstyle may be dont_touch/keep/- or 0..3.
- Omitted api_key/GDrive fields keep existing values.

/edit [language] [forgejo] [api_key] [gdrive_*] [wrapstyle] [announcement_channel]
- Rank 2. Edits existing server meta only. Omitted fields keep current values.
- Text fields use "-" to clear. announcement_channel:true stores the current
  channel id. GDrive completeness and wrapstyle rules match /configure.

/addapi key_name:<name> token:<value>
- Rank 2. Upserts a global env.pandora key. key_name may not contain newline,
  carriage return, or |; token may not contain newline.

/gentoken [label] [local]
- Rank 3. Generates a 64 hex character API bearer token using getrandom and
  appends it to API_TOKENS_PATH. Label is stored as a comment.
- local:true writes token|local|<guild_id>, binding the token to this server for
  Drive credentials and unlocking git console/API git endpoints.

/lstoken [page]
- Rank 3. Lists API bearer tokens by their first 3 and last 3 characters, label,
  and local binding state. page selects a 10-token page.

/rmtoken [label:<label>] [token:<abc...xyz>]
- Rank 3. Removes every API bearer token whose stored label exactly matches the
  supplied label, or removes one token whose displayed first3...last3 mask
  matches token. Exactly one of label/token is required.

/lsauth level:<level>
- Rank 3. Lists users in the selected auth level as <@ID> mentions. Supports
  authorize.pandora, fansubber.pandora, admin.pandora, upper.pandora, and
  witch.pandora.

/changerank command:<name> rank:<0-4>
- Rank 4. Updates DB/config/global/environment/command_ranks.pandora for a
  known command. It rejects attempts to change changerank itself.

/hearts
- Rank 2. Sends a HalfJob to the worker and edits the response with each shrine
  layer's alive/dead state, last heartbeat age, and reboot count.

/gitsync
- Rank 3. Sends a HalfJob to kill shrine layers, git pull the configured repo,
  optionally write PANDORA_GITSYNC_REQUEST, remove DB/work, and exit for restart.

/gettranslation language:<en|tr|jp> key:<key>
- Rank 2. Initializes language files if needed, uppercases the key, and replies
  ephemerally with text and arg count.

/addtranslation language:<en|tr|jp> key:<key> text:<text> [args]
- Rank 2. Upserts one localization entry in DB/config/<lang>.toml. Existing keys
  keep their current args unless args is provided; new keys infer args from {}
  when args is omitted.

/gettranslationall language:<en|tr|jp>
- Rank 2. Sends the selected localization TOML as an ephemeral attachment.

/addtranslationall language:<en|tr|jp> file:<toml>
- Rank 2. Downloads a UTF-8 .toml attachment, validates it as a non-empty map of
  MessageEntry values, serializes it pretty, and replaces DB/config/<lang>.toml.

/acixconfirm job_id:<id>
- Rank 4. Looks up the job DB and calls confirm_acix for a finished upload.
- Slash description begins with [BETA-TESTING].

/acixtemplate template:<id>
- Rank 4. Requires an attached channel and stores the AnimeciX fansub template id
  in channel meta.
- Slash description begins with [BETA-TESTING].

/font [file:<zip>] [link:<http_zip>]
- Rank 1. Requires either a zip attachment or http(s) zip link. Attachment wins
  when both are provided.
- Extracts the zip under DB/fontconfig/<server_id>. Unsafe paths are rejected.
- On Linux, recursively copies font files to /usr/local/share/fonts/pandora or a
  user font directory fallback and runs fc-cache when fonts were copied.

/readmebase file:<attachment>
- Rank 2. Stores the attachment bytes as DB/config/<server_id>/base.md.

/auth user_id:<id> [level]
- Rank 2 minimum, with target-level check. Appends the user id to the selected
  permission file, default authorize.pandora.

/remove user_id:<id> level:<level>
- Rank 2 minimum, with target-level check. Removes the user id from the selected
  permission file and reports whether it was present.

Legacy message and event entries
--------------------------------

!enc
- Rank 0. Replies telling the user to use /encode.

!ts <episode> with .ass or .zip attachment
- Rank 1. Legacy TS upload path. Requires attached channel and episode in range.
- Accepts .ass or exactly one root-level .ass in zip, loads with advanced parsing
  disabled, rewrites only the ASS title via libkagami dump, and uploads TS -
  <name> - E<NN>.ass with commit "Typeset". It does not use PNASS_JOB.

Cross reaction on a job response
- Sends a cancel half-job for the reacting user, channel id, and message id.

Help component id pnhelp:<owner>
- Updates the ephemeral help message only for the owner that opened /help.

Guild/channel/thread cache events
- cache_ready, guild_create, channel_create/update/delete, and thread_create/delete
  sync DB/config/<server>/channels.json. Channel/thread delete also auto-detaches
  channel meta without deleting the repo.

Removed
-------

- /scrape and handle_scrape are not present.

HTTP web entries
----------------

GET /
- Public. Serves the baked desktop shell HTML.

GET /encode
- Public. Serves the baked encode console HTML.

GET /git
- Public. Serves the baked git console HTML. Git API operations it uses still
  require a local bearer token.

GET /favicon and GET /favicon.ico
- Public. Serves the first DB/config/global/favicon.<png|ico|svg|jpg|jpeg|webp|gif>
  override found, otherwise the bundled web/favicon.png.

GET /health
- Public. Returns "ok".

HTTP API entries under /api/v1
------------------------------

GET /jobs?status=<active|ongoing|recent>
- Authenticated. status=ongoing returns non-terminal non-archived jobs,
  status=recent returns the 50 most recent archived jobs, anything else returns
  all non-archived jobs.

GET /jobs/:id
- Authenticated. Returns one JobStatus by id or 404.

POST /jobs/encode
- Authenticated, rate-limited. Body: torrent, subtitle_b64, optional preset,
  lang, channel_id, server_id. Decodes subtitle and queues an API encode job.
  Local tokens override requested server_id.

POST /jobs/backup
- Authenticated, rate-limited. Body: torrent, optional lang, channel_id,
  server_id, all. Queues Backup unless all=true, then BackupAll. Local tokens
  override requested server_id.

POST /jobs/probe
- Authenticated, rate-limited. Body: torrent. Queues an API probe job with
  channel_id 0, lang EN, and no server id.

POST /jobs/pancode
- Authenticated, rate-limited. Body: probe_job_id string, file_index,
  subtitle_b64, optional preset.
- Validates that the probe job exists, is active, and is stage Probed. Queues a
  Pancode job and persists progress metadata with probe id and file index.

POST /jobs/gitcode
- Authenticated, rate-limited. Body: torrent, subtitle_url, optional preset.
- Rewrites GitHub blob URLs to raw, fetches subtitle with a 180 second timeout,
  and queues an API encode job.

POST /jobs/:id/cancel
- Authenticated, not rate-limited. Rejects missing/terminal jobs. Local tokens may
  cancel only encode jobs from their own server. Sends a cancel half-job and
  returns 202.

POST /jobs/:id/acix/confirm
- Authenticated, rate-limited. Calls confirm_acix for the DB job id and returns
  the AnimeciX result or a bad gateway error.

GET /git/attachments
- Authenticated local token required. Returns libpngit::list_attachments for the
  local server.

GET /git/channels
- Authenticated local token required. Returns DB/config/<server>/channels.json,
  or [] if missing/invalid.

GET /git/readmebase
- Authenticated local token required. Returns server base.md when present. If
  absent, returns global base.md or bundled README_BASE_GUIDE with is_guide=true.

POST /git/readmebase
- Authenticated local token required, rate-limited. Body: content. Writes
  DB/config/<server>/base.md and returns saved/bytes.

POST /git/init
- Authenticated local token required, rate-limited. Body: mal, channel_id string,
  optional season, tl, tlc, ts, qc. Runs libpngit::init_repo and returns repo
  outcome JSON.

POST /git/attach
- Authenticated local token required, rate-limited. Body: mal, repo, channel_id
  string, optional season, tl, tlc, ts, qc. Runs libpngit::attach_repo and
  returns repo outcome JSON.

POST /git/source
- Authenticated local token required, rate-limited. Body: channel_id string,
  episode, link. Runs libpngit::set_source and returns written path/content.

POST /git/detach
- Authenticated local token required, rate-limited. Body: channel_id string.
  Removes channel attachment and returns name/repo_url.

POST /git/destruct
- Authenticated local token required, rate-limited. Body: channel_id string.
  Deletes the attached repo and returns name/owner_repo.

POST /git/smartcode
- Authenticated local token required, rate-limited. Body: channel_id string,
  episode, optional link and preset.
- Runs libpngit::smartcode_merge, queues an API encode against the merged ASS,
  stores GDrive folder overrides on the job, and returns job id, link, repo,
  release/source paths, folder ids, and warnings.
- If pnass produces no Dialogue lines, the route returns an error before writing
  the release ASS.

POST /gitsync
- Authenticated with an API token labelled PNwitch, rate-limited. Queues an API
  gitsync half-job and returns an accepted job id.

GET /acix/search
- Authenticated. JSON body: q and optional limit. Uses AnimeCix env config to
  search titles. Default limit is 20.

POST /acix/tmdb
- Authenticated, rate-limited. Body: tmdb_id and optional media_type. Converts a
  TMDB id to AnimeciX id; media_type movie/movies selects Movies, otherwise
  Series.

GET /acix/translators
- Authenticated. Returns AnimeciX translators from env-configured client.

POST /acix/publish
- Authenticated with an API token labelled PNwitch, rate-limited. Body: extra,
  url, template, title_id, optional season_num, episode_num. Calls AnimeCix
  multishare_mixed.
